# tg-ws-proxy-rs

[English version](README.en.md)

MTProto-прокси для Telegram, работающий через WebSocket. Написан на Rust.

Основан на [tg-ws-proxy](https://github.com/Flowseal/tg-ws-proxy) от Flowseal.

Большая часть кода написана с помощью Claude

## Как это работает

Клиент Telegram подключается к прокси по протоколу MTProto. Прокси шифрует трафик и пересылает его на серверы Telegram через WebSocket-соединения. Это позволяет прокси работать в сетях, где прямые подключения к Telegram заблокированы, но WebSocket-трафик проходит свободно.

Основные возможности:
- **WebSocket-туннелирование** -- MTProto-трафик оборачивается в WebSocket-фреймы и передается через стандартный HTTPS (порт 443)
- **Пул соединений** -- заранее установленные WebSocket-соединения к DC Telegram снижают задержку
- **Системный трей** -- работает в фоне с иконкой в трее (Windows/Linux/macOS), показывает статистику подключений, копирование ссылки, автозапуск

## Установка

### Скачать готовый файл

Перейдите в [Releases](../../releases) и скачайте файл для вашей платформы.

### Сборка из исходников

Требования: [Rust](https://rustup.rs/) 1.75+

```bash
git clone https://github.com/akuto/tg-ws-proxy-rs.git
cd tg-ws-proxy-rs
cargo build --release
```

Готовый файл будет в `target/release/proxy-app` (или `proxy-app.exe` на Windows).

## Использование

### Графический режим (по умолчанию)

Просто запустите файл. Прокси запустится в системном трее с автоматически сгенерированным секретом. По правой кнопке мыши на иконке в трее можно:
- Открыть ссылку прокси в Telegram
- Скопировать ссылку `tg://` в буфер обмена
- Включить/выключить автозапуск (Windows)
- Включить/выключить логирование ошибок
- Посмотреть статистику подключений

### Консольный режим

```bash
proxy-app --no-tray
```

### Параметры

```
  -p, --port <PORT>          Порт прослушивания [по умолчанию: 1443]
      --host <HOST>          Адрес прослушивания [по умолчанию: 127.0.0.1]
      --secret <SECRET>      Секрет, 32 hex-символа (генерируется автоматически)
      --dc-ip <DC:IP>        Перенаправление DC, напр. 2:149.154.167.220
      --no-tray              Консольный режим без системного трея
      --pool-size <N>        Размер пула WebSocket-соединений [по умолчанию: 4]
      --buf-kb <KB>          Размер буфера в КБ [по умолчанию: 64]
  -v, --verbose              Подробное логирование (консольный режим)
      --setup                Открыть ссылку прокси в Telegram и выйти
```

### Примеры

Запуск на всех интерфейсах с другим портом:
```bash
proxy-app --no-tray --host 0.0.0.0 --port 8443
```

Использование определенного секрета:
```bash
proxy-app --no-tray --secret 0123456789abcdef0123456789abcdef
```

### Конфигурация

При первом запуске рядом с исполняемым файлом сохраняется `config.json`. В нем хранится секрет и настройки, чтобы они сохранялись между перезапусками.

## Поддержка платформ

| Платформа | Иконка в трее | Автозапуск | Примечания |
|-----------|---------------|------------|------------|
| Windows   | Да            | Да (реестр) | В release-сборке работает как оконное приложение |
| Linux     | Да            | Нет        | Требуется поддержка системного трея (KDE, GNOME с расширением) |
| macOS     | Да            | Нет        | Работает со стандартной панелью меню |

На всех платформах режим `--no-tray` работает без каких-либо GUI-зависимостей.
